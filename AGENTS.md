# 项目工作入口

本项目的产品与工程纲领在 [`纲领.md`](./纲领.md)。

后续在本项目中新开的对话、代码修改、测试规划和验收判断，都应先读取并遵循 `纲领.md`：

- 以 Android 盲人 AI 无障碍助手 V1 为目标。
- 保持 V1 克制，不引入全天候热词唤醒、离线大模型或复杂自动化流程。
- 不在 Android App 内硬编码云端 AI API key，统一通过后端代理接入。
- 对无障碍、语音、截图、AI 动作执行相关改动，优先满足纲领中的阶段目标、动作白名单、安全确认和验收标准。

## 当前 APP 约束

- 本项目内后续所有 APP 层面的功能增加、功能删减、界面调整和行为修改，都必须基于现有的 SightSync Android APP 继续改动。
- 不要另起新的 APP、不要新建替代性 Android 应用模块，除非用户明确要求拆分或新建。
- 每次改动 SightSync Android APP 后，都必须在本机安卓虚拟机中运行验证功能是否正常、是否存在 bug；如果发现 bug，必须立即 debug 并修复后再次验证。
- 需要接入 AI 服务时，不得自行选择或接入 AI provider/model；必须先由用户决定接入什么 AI，并由用户提供对应 API 密钥。
